# 出了问题怎么办？一套排错思路帮你搞定

用 OpenClaw，就像用任何工具一样，偶尔肯定会出问题。出了问题不要慌——绝大多数问题都是配置的小问题，按照这个思路一步步查，你自己就能解决 90% 的情况。

这一章我帮你整理了一套排错方法，加上 10 个最常见的问题和解决办法。

---

## 排错的核心思路：四步走

不管遇到什么问题，你按照这四步走，基本都能找到原因：

### 第一步：先看日志，找到错误是什么

日志就是 OpenClaw 的"工作记录"——它做了什么、哪一步出了错、错误原因是什么，全都记在日志里。

**日志在哪里看：**
- 如果你是后台运行的（daemon 模式）：日志在 `~/.openclaw/logs/` 目录下
- 如果你是在终端直接运行的：错误信息直接显示在终端里

> 💡 **daemon 模式是什么？** 就是"在后台不间断运行"的意思。你用 `openclaw onboard --install-daemon` 安装的时候就默认是这个模式——OpenClaw 在后台一直跑着，你关掉终端它也不停。

**看日志的技巧：**
- 在日志文件里搜索 `error` 或 `Error` 关键词
- 重点看**最后出错的那几行**，一般错误原因就写在那里
- 如果日志太长看不懂，**把最后 20 行复制出来**，后面搜索或者问社区都用得上

### 第二步：搜一搜，别人碰到过吗

你碰到的问题，99% 的可能性别人也碰到过。把错误信息的**关键句子**复制出来，去搜一下：

- 去 OpenClaw 的 GitHub Issues 页面搜（这里最精准）
- 去百度或 Google 搜，加上 `OpenClaw` 关键词
- GitHub 上经常能找到核心开发者的回答，跟着改就行

### 第三步：对照检查配置

很多问题其实根本原因就四个字：**配置写错了**。你按这个清单检查：

**配置检查清单：**

- [ ] JSON 格式对不对？（有没有漏逗号、括号不匹配？）
- [ ] API Key 对不对？（有没有多复制了空格？少复制了字母？过期了？）
- [ ] 模型 ID 对不对？（是不是拼写错了？）
- [ ] baseUrl 对不对？（有没有少了 `https://`？末尾多了 `/`？）
- [ ] 改完配置重启了吗？（不重启不生效！）
- [ ] 权限开了吗？（飞书权限、API 权限都要开）

> 💡 **怎么检查 JSON 格式？** JSON 就是 OpenClaw 配置文件用的格式，它对**逗号和括号**非常严格——少一个逗号、多一个括号就会整个文件不能用。你可以把你的配置文件内容复制到网上的"JSON 校验工具"（搜一下"JSON validator 在线"就能找到），它会用红色标出你哪里写错了，一下子就能找到。

### 第四步：还是不行？去问社区

如果前三步都试过了还是搞不定，去 OpenClaw 的 GitHub 开一个 Issue 向社区求助。

**提问的正确姿势（这样问别人才好帮你）：**

1. 说清楚你的系统：macOS / Linux / Windows？什么版本？
2. 说清楚你做了什么：一步步描述你的操作
3. 说清楚你碰到了什么：复制完整的错误信息
4. 贴上日志：把相关的日志贴出来

写得越清楚，别人帮你越快。千万不要只写一句"不能用了求助"——这样谁也帮不了你。

## 10 个最常见问题和解决方法

### ❓ 1. 安装完了，运行 `openclaw` 提示"command not found"

**原因：** 安装目录没有加到系统的 PATH 里面。

> 💡 **PATH 是什么？** 你在终端里输入一个命令名字（比如 `openclaw`），系统需要知道这个程序文件在哪里。PATH 就是一个"地址列表"，系统会按照这个列表去一个一个文件夹里找。如果 openclaw 装在的文件夹不在这个列表里，系统就找不到它，就会说"command not found"。

**解决办法：**
1. 先试试：关掉终端重新打开，再输 `openclaw --version`
2. 如果还是不行，运行 `npm config get prefix`，看看 npm 的全局安装目录在哪里
3. 把那个路径加到你的 PATH 里（搜一下"macOS 添加 PATH"或"Linux 添加 PATH"就有教程）

---

### ❓ 2. 模型调用失败，报错"401 Unauthorized"

**原因：** API Key 不对，或者已经过期了。

**解决办法：**
1. 去你的大模型厂商后台，检查 API Key 是否还有效
2. 重新复制一遍——小心不要多复制了空格或少复制了字母
3. 检查是不是余额用完了，需要充值

---

### ❓ 3. 飞书机器人发消息收不到回复

**原因：** 通常是配置步骤漏了。

**解决办法，按顺序检查：**
1. 你发布版本了吗？没发布飞书收不到消息
2. 你配置事件订阅了吗？选了"长连接"吗？
3. 你把 App ID 和 App Secret 填对了吗？
4. 你改完配置重启 OpenClaw 了吗？
5. 你配对了吗？第一次用必须配对

---

### ❓ 4. 配置文件改了不生效

**原因：** 改完配置一定要重启 OpenClaw 服务。

**解决办法：** 重启服务。如果是 daemon 模式，运行：
```bash
openclaw daemon restart
```

---

### ❓ 5. 技能安装了但是 AI 不调用

**原因：** 可能有几个原因。

**解决办法：**
1. 检查技能有没有正确安装在 `skills/` 目录下
2. 看看技能的 `SKILL.md` 文件，有没有要求配置环境变量（比如 API Key）
3. 重启 OpenClaw 让它重新加载技能
4. 试着在对话中直接提到你要用的功能，比如"用 tavily 搜索一下 XXX"

---

### ❓ 6. 回复很慢或者超时

**原因：** 通常是网络或模型的问题。

**解决办法：**
1. 检查你的网络是不是正常的
2. 如果用海外模型（如 OpenAI），检查你的网络连接是否能正常访问海外服务
3. 换一个模型试试——在聊天里输入 `/model` 切换
4. 在配置里设置 `fallbacks`（备用模型），主模型超时了自动切换

> 💡 **fallbacks 是什么？** 就是"备胎"——你可以在配置里写上两个模型，主力模型要是挂了或者回复太慢，OpenClaw 会自动切换到备用模型，这样你的 AI 就不会突然"死机"了。

---

### ❓ 7. JSON 配置文件格式出错了，OpenClaw 启动不了

**解决办法：**
1. 把 `~/.openclaw/openclaw.json` 的内容复制到在线 JSON 校验工具
2. 找到标红的错误位置，通常是少了逗号或者括号不配对
3. 修好之后保存，重启就行

> 💡 小技巧：改配置之前**先备份一份**原始文件。在终端里运行：`cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak`（`cp` 就是复制命令，`.bak` 是备份的意思），这样改坏了也能恢复。

---

### ❓ 8. Gateway 认证失败（新版本常见）

**原因：** OpenClaw 新版本加强了 Gateway 安全，首次启动需要认证。

**解决办法：**
1. 运行 `openclaw gateway auth` 完成认证
2. 如果提示需要 token，在 WebUI 控制面板里找到 Gateway token 复制过来
3. 如果还是不行，看看是不是需要更新到最新版本：`npm update -g openclaw@latest`

---

### ❓ 9. 定时任务没有按时执行

**解决办法：**
1. 检查你的 cron 时间格式对不对（记住：分钟在前，小时在后）
2. 确认 OpenClaw 服务一直在运行（daemon 模式不能停）
3. 确认你的电脑没有休眠——电脑休眠了后台程序也会暂停
4. 检查日志看定时任务有没有触发记录

---

### ❓ 10. 想恢复出厂设置，重新来

如果你觉得改来改去弄乱了，想从头来过：

```bash
# 备份你的现有配置（以防万一）
cp -r ~/.openclaw ~/.openclaw.backup
```

> 💡 **cp -r 是什么？** `cp` 是复制命令，`-r` 表示"连文件夹里的所有东西都复制"。这行命令就是把你整个 OpenClaw 配置文件夹复制一份到 `.openclaw.backup`，万一重置后后悔了，还能把旧的恢复回来。

```bash
# 重新初始化
openclaw onboard --install-daemon
```

这样就从头开始配置了。你之前的配置保存在 `.openclaw.backup` 里面，想恢复随时都能恢复。

---

## 记住：排错思路比具体命令重要

OpenClaw 更新很快，今天的具体命令明天可能就变了，但是排错思路永远不变。你只要记住这四步：

```
看日志 → 搜一搜 → 查配置 → 问社区
```

按这个顺序来，绝大多数问题你自己就能解决。实在搞不定也别怕，OpenClaw 社区很热心，提个 Issue 基本都有人帮你。
