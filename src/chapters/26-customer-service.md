# 客户服务与自动回复系统

前面第 20 章我们提到过 Airbnb 房东助手的案例。这一章我们把这个方向深入展开——教你用 OpenClaw 搭建一个通用的**客户服务 / 自动回复系统**。

不管你是做民宿的、开网店的、做咨询的，还是任何需要回答重复问题的场景，这一章都能帮到你。

---

## 核心思路：FAQ → 自动回复 → 转人工

一个好的客服系统有三层：

1. **第一层：FAQ 自动回答**——客户问的 80% 的问题都是重复的（价格多少？怎么退货？WiFi 密码是什么？），这些 AI 直接回答
2. **第二层：智能判断**——AI 判断这个问题自己能不能回答，不能的就转给你
3. **第三层：客户画像**——AI 记住每个客户的历史交流，下次来了就知道他的情况

---

## 第一步：建立你的 FAQ 知识库

把你常被问到的问题整理成一份文件。**照着下面的步骤做：**

**1. 创建 knowledge 文件夹**（如果你跟着第 23 章做过了就跳过这一步）：

```bash
mkdir -p ~/.openclaw/knowledge
```

**2. 用 VS Code 创建 FAQ 文件：**

```bash
code ~/.openclaw/knowledge/faq.md
```

> 💡 **`code` 命令打不开？** 用 Finder 打开 `~/.openclaw/knowledge/` 文件夹（记得按 `Command + Shift + .` 显示隐藏文件），右键新建文件，命名为 `faq.md`，再用 VS Code 打开。

**3. 把下面的模板复制粘贴进去**，然后改成你自己的内容：

```markdown
# FAQ 常见问题

## 价格与付费
Q：你们的产品多少钱？
A：我们有三个版本：基础版 299/月，专业版 599/月，企业版请联系销售。

Q：可以试用吗？
A：可以，注册后有 14 天免费试用期，期间功能全开放。

Q：怎么退款？
A：购买后 7 天内可以全额退款，联系客服即可。

## 使用问题
Q：怎么注册？
A：打开我们的官网，点击右上角"免费注册"，用手机号或邮箱注册即可。

Q：app 打不开怎么办？
A：请先检查网络连接。如果网络正常仍然打不开，请卸载重装最新版本。

# 更多问题请联系人工客服
```

> ⚠️ **关键：把上面的内容全部换成你自己的业务问题。** 想想客户平时问你最多的 10 个问题是什么，按照 `Q：` `A：` 的格式写进去就行。

**4. 保存文件。**

---

## 第二步：写一个客服技能

**1. 创建技能文件夹：**

```bash
mkdir -p ~/.openclaw/skills/customer-service
```

**2. 用 VS Code 创建技能文件：**

```bash
code ~/.openclaw/skills/customer-service/SKILL.md
```

**3. 把下面的内容复制粘贴进去**，保存：

```markdown
---
name: "customer-service"
description: "自动回答客户常见问题，处理不了的转人工"
tags: ["客服", "自动回复"]
---

# 客服助手技能

## 触发条件
当收到客户消息时自动触发

## 执行步骤
1. 先在 FAQ 知识库中搜索是否有匹配的问答
2. 如果找到匹配 → 直接回答客户，语气友好专业
3. 如果没找到匹配 → 尝试根据已有知识回答
4. 如果完全无法回答 → 回复："感谢您的提问，这个问题我需要转给专业同事处理，请稍等。" 同时通知主人

## 回复原则
- 语气：友好、专业、不啰嗦
- 回答要简洁，不超过 3 句话
- 涉及价格、退款等敏感话题，严格按 FAQ 回答，不要自由发挥
- 任何投诉类问题，一律转人工

## 注意事项
- 绝对不要编造不确定的信息
- 不要做超出权限的承诺（如"保证解决"等）
```

**4. 保存文件后**, 你可以打开 OpenClaw 聊天窗口测试一下：

> "假装你是一个客户，问我'你们的产品多少钱'"

AI 应该会根据你 FAQ 里的内容来回答。如果回答正确，说明技能和 FAQ 都配好了。

---

## 第三步：设置通知与转人工

当 AI 遇到处理不了的问题时，可以通过飞书通知你：

在 `USER.md` 里加上一条规则：

```markdown
## 客服转人工规则
当你遇到以下情况时，发飞书消息通知我：
- 客户投诉
- 涉及退款金额超过 500 元
- 你不确定怎么回答的问题
- 客户连续问了 3 个问题你都答不上来
通知格式：[转人工] 客户XXX的问题是：XXX
```

---

## 第四步：客户画像（进阶）

利用 OpenClaw 的记忆系统，AI 会自动记住每个客户的交流历史。你可以主动让它建画像：

> "帮我整理一下客户张三的所有交流记录，生成一份客户画像：他问过什么问题、买过什么、有什么偏好、有没有投诉过"

这样下次张三再来，AI 就知道他是老客户了，能更有针对性地回应。

## 实际案例：小型电商客服

一个做手工皂的小老板分享了她的用法：

1. **FAQ**：产品成分、发货时间、退换货政策
2. **自动回复**：客户问发货时间 → AI 自动回答"下单后 48 小时内发货"
3. **商品推荐**：客户说"我皮肤比较干" → AI 根据 FAQ 推荐保湿款
4. **转人工**：客户要定制 → AI 转给她本人处理
5. **每周汇总**：每周 AI 自动总结"本周客户问得最多的 5 个问题"——她根据这个更新 FAQ

她说："我以前每天花 2 小时回消息，现在 30 分钟就搞定了。"

---

## ⚠️ 注意事项

1. **一定要有转人工机制**：不管 AI 多聪明，总有它处理不了的情况。没有转人工 = 客户流失
2. **定期更新 FAQ**：客户的问题会变的，每个月更新一次 FAQ
3. **不要让 AI 做决策**：价格折扣、退款审批这些，必须你来决定，AI 只负责传达
4. **保护客户隐私**：不要在 FAQ 或技能文件里放客户的个人信息

---

## 小结

| 层级 | 功能 | 实现方式 |
|------|------|---------|
| 第一层 | FAQ 自动回答 | knowledge/faq.md |
| 第二层 | 转人工通知 | 飞书消息通知 |
| 第三层 | 客户画像 | OpenClaw 记忆系统 |

**AI 客服的目标不是替代你，而是帮你过滤掉 80% 的重复问题，让你把时间花在 20% 真正需要你的客户身上。**
