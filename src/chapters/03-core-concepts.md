# 五个核心组件，用大白话讲明白

上一章我们说了，AI 智能体就是能自己帮你干活的 AI。那它到底是怎么做到的呢？其实说穿了很简单，就是**五个核心组件**拼起来的。

这一章我一个个给你讲，每个都用生活化的类比，保证你看完就明白智能体到底怎么工作，一点都不深奥。

## 第一个组件：推理服务 = AI 的大脑

我给你打个比方：推理服务就像是我们人的大脑，你遇到问题，大脑思考，然后张嘴回答。就这么简单。

我再给你说清楚一点：大模型本质就是一个超大的参数文件，存在你的硬盘上，你不加载它，它就是一堆死数据，没法用。**推理服务就是把这个大文件加载到显卡内存里，你提问，它算，然后一点点给你吐出回答。**

就是这么简单：
- 你提问 → 推理服务计算 → 给你输出回答
- 没了

你其实不用自己搭推理服务，现在大多数人都是用 OpenAI、Anthropic 或者国内厂商提供的服务，你注册账号充点钱就能用，人家都给你准备好了，你拿来就能用。

## 第二个组件：记忆机制 = AI 的笔记本

你可能会想，AI 不是什么都记得吗？怎么还要记忆机制？

其实真不是这样。我给你解释一下：

推理服务每次处理你的问题都是"无状态"的——什么意思呢？就是它处理完你的问题，就把这件事忘了，下次你再来问，它还是什么都不知道。

而且还有一个限制：大模型一次能处理的文字字数是有限的——这个限制叫做**上下文窗口**（context window），就那么大一个箱子，你不可能把一年前说过的话都塞进去。举个例子，现在主流模型的上下文窗口大概能容纳 10-20 万字，听起来不少，但你要是把过去半年的聊天记录都塞进去，肯定塞不下，塞下了算起来也特别贵。

所以需要一套"记忆机制"来管理这些东西，就像你记笔记一样：

- **短期记忆**：最近几轮对话，原封不动保留着，保证这次对话能连贯，你说"刚才那个问题改一下"，它知道你改哪个。比如，你问完"帮我总结一下这份报告"，接着说"把总结的第三点再详细展开"，AI 就能知道你指的是哪份报告和哪一点。
- **长期记忆**：时间久了的对话，会用一个小模型提炼一下，把重要的留下来，比如"用户喜欢简洁回答"、"用户住在上海"，存在文件里，下次要用的时候再拿出来。比如，AI 记住你上次提过喜欢用 Markdown 格式，下次它就会自动用 Markdown 给你排版。

这样一来，AI 就真的能"记住你"，用得越久，它越懂你。这个设计真的很聪明。

## 第三个组件：RAG 检索增强 = AI 的参考书

RAG 全称是 Retrieval-Augmented Generation，翻译过来就是"检索增强生成"——你不用记这个名字，就记住它干的事就行：**你写文章遇到不懂的，先翻参考书，看完再写，RAG 干的就是这个。**

大模型有个天生的问题：它的知识来自训练数据，训练完就"冻住"了，之后发生的新闻它不知道，你存在自己电脑上的私人文件它也看不到。

所以 RAG 就是解决这个问题的，流程就是：

1. 你问了一个问题
2. AI 先去资料库里搜，找到和这个问题相关的内容
3. 把搜到的内容拼到你的问题里，一起给大模型
4. 大模型参考这些内容回答你

这样回答就准确多了，不会一本正经地胡说八道。

比如你问它"我去年做那个项目进度怎么样了"，它先在你的记忆里找到那个项目的记录，拼给大模型，大模型就能准确回答你了。就是这么简单。

## 第四个组件：MCP 工具调用 = AI 的手

你想想，有了大脑（推理）、笔记本（记忆）、参考书（RAG），AI 还是只能待在服务器里边说说话，没法真的动手改变东西啊对不对？

MCP（全称 Model Context Protocol，模型上下文协议）就是给 AI 添上一双手，让它能自己动手干活。MCP 是一套规矩，告诉 AI 怎么调用外面的工具：

- 你要搜索网络 → 调用搜索工具
- 你要读文件 → 调用文件读取工具
- 你要发邮件 → 调用邮件工具
- 你要运行命令 → 调用终端工具

有了这双手，AI 才能真的动手干活。MCP 就是一套统一的规矩，不管是什么工具，AI 都能用同样的方式调用，特别方便。

## 第五个组件：Skills 技能 = AI 的菜谱

MCP 解决了"AI 能不能调用工具"的问题，但是还有一个问题：AI 不知道**什么时候用什么工具，按什么顺序用**。

就像是你准备好了所有食材和锅铲，你还是需要一本菜谱告诉你，第一步放什么第二步放什么，放多少盐，对不对？

Skills 就是菜谱：它把特定场景下，**工具该怎么用，第一步做什么第二步做什么，要注意什么**，都写好了，AI 照着做就行，不会错。

我给你举个例子：
- 处理客服工单：先读用户的诉求 → 查用户历史记录 → 给解决方案 → 解决不了转人工 → 关闭工单
- 整个流程就是一个 Skill，AI 照着走就不会错

Skills 还有一个特别好的设计叫"渐进式加载"：你不用的时候不加载，你用哪个技能加载哪个，不会一下子把几百个技能都塞给 AI，这样 AI 不会糊涂，调用起来更准确。真的很聪明。

## 什么时候就变成完整的 AI 智能体了？

我给你整理一下，当一个 AI 同时有这五个组件，它就是一个能自主行动完成目标的 AI 智能体了：

| 组件 | 干什么 |
|------|--------|
| 推理服务 | 负责思考回答 |
| 记忆 | 记住你的偏好和历史对话 |
| RAG | 找不到答案先翻参考书 |
| MCP | 伸出手调用工具干活 |
| Skills | 按菜谱一步步做事 |

就是这么回事，一点都不深奥对不对？
