# ClawHub 技能商店：怎么找、怎么装、怎么选

ClawHub 就是 OpenClaw 的官方技能商店——就像你手机上的应用商店一样。社区已经贡献了超过 **10,000 个技能**，覆盖了办公效率、内容创作、数据分析、自动化等多个大类。

你需要什么功能？搜一下，一条命令就装上了。这一章我教你从头到尾怎么用。

## 第一步：确保 clawhub 工具可用

如果你已经安装了 OpenClaw，clawhub 命令行工具已经一起装好了。你可以验证一下：

```bash
clawhub --version
```

能输出版本号就行了。如果提示找不到命令，装一下：

```bash
npm install -g clawhub
```

## 怎么搜索技能？

### 方法一：在终端搜索

最简单的方法，输入关键词就行：

```bash
clawhub search 你的关键词
```

或者你也可以用 `find-skills` 命令（效果一样，只是名字不同）：

```bash
find-skills 你的关键词
```

比如你想找 PDF 相关的技能：

```bash
clawhub search pdf
```

它就会列出所有匹配的技能，包括名称、简介、下载量和社区评分。

### 方法二：直接问 AI 帮你搜（更方便）

如果你装了 `find-skills` 技能，你直接在聊天里告诉 AI：

- "帮我找一个能处理 PDF 的技能"
- "有没有什么好用的日程管理技能？"
- "帮我搜一个关于邮件自动化的技能"

AI 会帮你搜 ClawHub，然后告诉你有哪些选择、评分怎么样、推荐哪个——比你自己敲命令还方便。

### 方法三：去 ClawHub 网站浏览

你也可以打开 ClawHub 的网站直接浏览，有分类导航，支持按下载量、评分排序。就像你逛应用商店一样，看看有什么有意思的技能。

## 怎么安装技能？

搜到了你想要的技能，记住它的名字（也叫 slug，就是技能的唯一标识名，一般是小写英文中间加横线，比如 `tavily-search`），然后一条命令安装：

```bash
clawhub install 技能名字
```

> 💡 **你可能在本书其他地方看到 `openclaw skills add 技能名字` 这种写法**——效果是一样的，都是安装技能，只是命令名字不同。用哪个都行，不用纠结。

比如安装 tavily-search：

```bash
clawhub install tavily-search
```

它会自动下载，安装到你当前工作区的 `skills/` 目录下面，装完就能用了。

> 💡 **技能安装到哪里了？** OpenClaw 找技能的时候会按这个顺序找：
> 1. 你当前项目的 `skills/` 文件夹（项目专用的）
> 2. `~/.openclaw/skills/` 文件夹（全局通用的）
> 3. OpenClaw 内置的技能
> 
> 所以你可以给不同的项目装不同的技能，互不干扰，挺灵活的。

## 安装完了怎么配置？

大多数技能装完就能直接用了。但是有些技能需要你**配置一下环境变量**（比如填入 API Key）。

> 💡 **环境变量是什么？** 你可以理解为"给程序传递配置信息的一种方式"。比如 tavily-search 需要你注册一个 Tavily 账号拿到 API Key，然后告诉 OpenClaw 你的 Key 是什么——这个"告诉"的过程就是设置环境变量。

怎么知道需不需要配置？打开技能文件夹里的 `SKILL.md` 看看就行，它会写清楚：

- 需要配置什么
- 去哪里拿 API Key
- 怎么设置

比如 tavily-search 的 `SKILL.md` 可能会写：

```
## 配置
1. 去 tavily.com 注册一个免费账号
2. 在后台获取你的 API Key
3. 设置环境变量：TAVILY_API_KEY=你的Key
```

你按照它说的做就行了。

### 怎么设置环境变量？

最简单的方式是在 OpenClaw 的配置文件里添加。或者你也可以在终端里设置：

```bash
# macOS / Linux
export TAVILY_API_KEY="你的API Key"
```

> 💡 **export 命令是什么意思？** `export` 就是在终端里设置一个“环境变量”——你可以把它理解为给程序贴了一张便利贴，上面写着"你的 API Key 是 XXX"。程序运行的时候会去看这张便利贴。不过注意，这个设置在你关掉终端后就会消失，所以建议你把它写进 OpenClaw 的配置文件里，这样就一劳永逸了。

设置完了**重启 OpenClaw** 让它识别，就搞定了。

## 怎么选技能？选好的不选多的

ClawHub 上面同一个功能可能有好几个技能，你该选哪个？其实很简单，看三样东西：

### 1. 看评分

ClawHub 上的技能有社区评分，满分 5 分。评分是大家用出来的，**选评分最高的那个准没错**。

### 2. 看下载量

下载量多说明很多人在用，出了问题也更容易找到帮助。

### 3. 看作者

知名作者、标有"官方认证"的技能质量更有保障。如果作者是匿名的、下载量个位数的，建议谨慎一点。

总结一下选技能的口诀：**评分高的选评分，下载多的选下载，认识的作者选作者。**

## 装了不需要了怎么删？

超级简单——直接把 `skills/` 文件夹下面对应的技能文件夹删掉就行。没有什么复杂的卸载命令，就是删个文件夹。

```bash
# 比如要删掉 weather 技能
rm -rf ~/.openclaw/skills/weather
```

> 💡 **rm -rf 是什么意思？** `rm` 是删除命令，`-r` 表示"连文件夹里面的所有东西都删"，`-f` 表示"不要问我确认直接删"。这个命令很强力，删了就没有了，所以**一定要确认你删的是对的文件夹**，别删错了。

删完了重启 OpenClaw，它就不会再加载这个技能了。

## 想自己写一个技能可以吗？

完全可以！而且比你想的简单。

你只需要在 `skills/` 文件夹下新建一个文件夹，然后在里面创建一个 `SKILL.md` 文件，用自然语言把"做这件事应该怎么做"写清楚就行了。

比如你想创建一个"日报生成"的技能：

1. 新建文件夹：`~/.openclaw/skills/daily-report/`
2. 在里面创建 `SKILL.md`：

```markdown
# 日报生成技能

## 触发条件
当用户要求生成工作日报时

## 步骤
1. 询问用户今天做了哪些事情（如果用户没主动说的话）
2. 按照以下格式整理成日报：
   - 今日完成事项
   - 遇到的问题
   - 明日计划
3. 语言简洁，每条控制在一行
4. 生成完后问用户要不要调整

## 格式示例
📋 工作日报 周五
✅ 今日完成：
- 完成了XX功能的开发
- 和客户确认了需求变更
🚧 遇到的问题：
- XX接口返回延迟较高
📌 明日计划：
- 优化XX接口性能
```

就这么简单，保存之后重启 OpenClaw，你就有了一个自己专属的日报技能。

## 小结

- **ClawHub 就是 OpenClaw 的应用商店**，10,000+ 技能等你挑
- **搜索**：`clawhub search 关键词`，或者直接问 AI
- **安装**：`clawhub install 技能名`，一条命令搞定
- **配置**：看 `SKILL.md` 里的说明，按要求设置环境变量
- **选技能**：评分高、下载多、作者靠谱
- **删除**：直接删文件夹
- **自己写**：新建文件夹+写 `SKILL.md`，用自然语言就行
