# 需求说明书：OpenClaw 小龙虾入门指南

## 项目定位

一本面向**零基础普通人**的 OpenClaw 入门电子书，帮助读者从"不知道什么是 AI 智能体"到"能独立搭建并使用自己的 AI 助理"。

## 目标读者

- 对 AI 感兴趣但没有技术背景的普通人（macOS 或 Windows 用户）
- 会基本的电脑操作（能打开终端、复制粘贴命令）
- 不需要会写代码

**不适合**：已经在用 OpenClaw 的开发者，想深入源码级分析的人。

## 写作原则

1. **双系统覆盖**：所有终端操作同时给出 macOS 和 Windows（PowerShell）版本
2. **一步一步照做**：关键操作给出编号步骤 + 可复制粘贴的命令 + 验证方法
3. **随时能逃课**：涉及配置文件修改的地方，提供"让 AI 帮你配"的逃课方法
4. **成功/失败判断**：每个操作后告诉读者"成功长什么样，失败长什么样"
5. **安全感**：反复强调"不会搞坏电脑"、"随时可以重置"

## 风格规范

| 规范 | 要求 |
|------|------|
| 语气 | 轻松口语化，像朋友聊天，不是教科书 |
| 术语 | 每个技术术语必须用大白话解释一遍 |
| 类比 | 多用生活化的比喻（菜谱、电话总机、笔记本等） |
| 结构 | 多用小标题、表格、列表，方便快速浏览 |
| 长度 | 每章 2000-4000 字，不啰嗦也不太短 |
| 💡 提示框 | 用 `> 💡` 解释容易困惑的概念 |
| ⚠️ 警告框 | 用 `> ⚠️` 标注安全提醒和新手易错点 |
| 双系统 | 所有终端命令同时给出 macOS 和 Windows 版本 |
| 逃课方法 | 配置操作提供"直接对 AI 说…让它帮你配"的替代方案 |

## 项目结构

```
openclawbook/
├── src/
│   ├── chapters/          ← 32 个 Markdown 章节源文件（00-toc ~ 31-appendix）
│   ├── images/            ← 截图和插图（待补充）
│   └── full-book.md       ← 构建合并后的完整书稿
├── docs/
│   ├── requirements.md    ← 本文件
│   └── research/          ← 研究资料和反馈
├── scripts/
│   └── build.sh           ← 一键构建脚本（merge / pdf / mobile）
├── templates/pdf/         ← PDF 排版模板
└── pub/pdf/               ← 生成的 PDF 文件
```

## 构建方式

```bash
# 合并所有章节
./scripts/build.sh merge

# 生成 A4 PDF
./scripts/build.sh pdf

# 生成手机 PDF
./scripts/build.sh mobile

# 全部构建
./scripts/build.sh all
```
