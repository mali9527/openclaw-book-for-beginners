# OpenClaw 入门书 · 插图需求文档

> **统一风格**：黑白中性笔手绘（详见 `docs/illustration-style-guide.md`）
>
> **管理规则**：
> - 插图文件统一存放在 `src/images/{章节编号}/` 目录
> - 文件名使用小写英文+连字符，如 `five-components.png`
> - 在章节 Markdown 中使用相对路径引用：`![描述](../images/XX/filename.png)`
> - 本文档是插图的"单一数据源"，增删改插图均从此文档开始
>
> ⚠️ **同步维护规则**：
> - **章节内容发生修改时，必须同步更新本文档中对应插图条目的位置信息**（行号、段落标题）
> - 每条插图记录的 `位置` 字段包含两部分定位：①段落标题（如 `## 举个例子`）和 ②约行号。段落标题是主定位依据，行号是辅助参考
> - 如果章节内容增删导致行号偏移，应在本文档中更新行号
> - 如果章节结构调整（标题变更、段落合并/拆分），应同步更新段落标题和行号
