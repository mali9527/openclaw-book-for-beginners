# 技能是什么？为什么说它让 OpenClaw 无限扩展

很多人刚用 OpenClaw 会问：技能到底是什么？为什么说技能是 OpenClaw 的灵魂？

这一章我们讲透这个问题，看完你就明白了。

## 一句话说清楚技能

**技能 = 教 AI 怎么做一件特定事情的"菜谱"。**

我们之前说过：

- MCP 解决了"AI 能不能调用工具"的问题
- 但 AI 不知道"什么时候用工具，按什么顺序用"

就像：你有了锅、菜、调料，你还是需要一本菜谱告诉你，先放什么后放什么，大火小火多久，不然你还是做不出好菜。

技能就是菜谱：它明确告诉你，做这个特定任务，**第一步做什么，第二步做什么，用什么工具，注意事项是什么**。AI 照着做就行，不会错。

## 举个例子你就懂了

比如你想要 AI 帮你搜索最新新闻，然后总结。

如果没有技能：

- AI 可能会忘记搜索，直接瞎编
- 可能搜索了忘记总结
- 可能顺序错了，先总结再搜索

有了搜索技能：

- 第一步：调用 tavily 搜索工具，搜索关键词
- 第二步：拿到搜索结果
- 第三步：把搜索结果整理总结，给用户看
- 顺序对，步骤对，不会漏

你看，技能就是把"怎么做"写清楚，AI 照着走就能做对。

## 渐进式加载：为什么这个设计好

OpenClaw 技能有一个很好的设计叫**渐进式加载**：

- 你不用的时候，不加载，不占 token
- 你要用这个技能的时候，才加载技能的内容进去
- 这样哪怕你装了 100 个技能，也不会一下子把所有技能都塞给 AI，不会占满上下文，AI 调用更准确

这个设计真的很聪明，解决了"技能装多了 AI 就糊涂"的问题。

## 技能从哪里来？ClawHub 技能商店

OpenClaw 官方有一个技能商店叫 **ClawHub**，类似于 npm 对于 Node.js，上面已经有上千个社区贡献的技能了。

你可以：

- 搜索你需要的技能
- 一条命令安装
- 用完了不用可以删掉

不用自己写，直接用社区做好的，太方便了。

当然，你也可以自己写技能，分享给别人，或者自己用。

## 技能都能帮你做什么？

各种各样的事情都有技能：

- 搜索网络：`tavily-search`
- 处理 PDF：`nano-pdf`
- 操作 GitHub：`github-cli`
- 天气查询：`weather`
- ...... 几乎你能想到的，社区都有人做了

你需要什么功能，搜一下，装了就能用，不用自己写代码，对普通人太友好了。

## 为什么说技能是 OpenClaw 的灵魂？

因为：

- OpenClaw 本身只是一个平台
- 技能给了 OpenClaw 真正的能力
- 你需要什么能力，装对应的技能就行
- 技能可以无限加，所以 OpenClaw 的能力可以无限扩展

没有技能，OpenClaw 就是一个空架子；有了技能，它才能真的帮你干活。所以说技能是灵魂。

## 小结

- 技能 = 菜谱，告诉 AI 做一件事的步骤和工具顺序
- MCP 给了你工具，技能给了你菜谱
- 渐进式加载：用哪个加载哪个，不浪费 token
- ClawHub 技能商店：现成技能上千个，一条命令安装
- 技能无限扩展，所以 OpenClaw 能力无限，这就是为什么技能是灵魂
