# 出了问题怎么办？排错思路比具体命令重要

用 OpenClaw，就像用任何复杂工具，偶尔肯定会出问题。出问题不要慌，按照这个思路一步步查，90% 的问题你自己就能解决。

## 第一步：先看日志

OpenClaw 有完整的日志，绝大多数问题，日志里面都写清楚了。

**日志在哪里看：**
- 如果你是 daemon 模式运行，日志在 `~/.openclaw/logs/` 下面
- 如果你是终端直接运行，输出就在终端里面

**看日志看什么：**
- 找 `error` 或者 `Error` 关键字，看错误说明是什么
- 看看在哪一步出的错，是安装错了还是配置错了还是运行错了

**记住：日志就是最好的老师，它告诉你的错误比我告诉你的清楚多了。**

## 第二步：搜一搜

拿到错误信息，复制关键句子，去 GitHub 搜一搜，去百度搜一搜，你碰到的问题别人肯定也碰到过，大部分情况都能找到解决方法。

搜索技巧：
- 直接复制错误的关键句子，加上 `OpenClaw` 关键词一起搜
- 能搜到 issue 或者教程，跟着改就行了

## 第三步：检查配置

很多问题其实都是配置错了，你对照这几点检查：

### 常见配置错误：
1. **JSON 格式错了** - 是不是漏了逗号，括号配对不对？你可以把你的配置复制到网上的"JSON 校验工具"查一下，一下子就找到错在哪里
2. **API Key 错了** - 是不是多复制了空格？少复制了字母？重新复制一遍试试
3. **模型 ID 错了** - 对照厂商文档看看 ID 对不对
4. **权限错了** - API Key 有正确权限吗

## 如果还是不行，去问社区

如果你搜了还是解决不了，去 OpenClaw GitHub 开一个 Issue，把你的错误日志贴上去，社区有人帮你。

问问题记得：
- 说清楚你是什么系统（macOS/Linux/Windows）
- 说清楚你做了什么，问题是什么
- 贴上错误日志
- 这样别人一下子就能帮你找到问题

## 记住：方法比具体命令重要

OpenClaw 更新快，今天的具体命令明天可能就变了，但是排错思路永远不变：

1. **先看日志** → 错误是什么
2. **再搜索** → 别人碰到过解决了吗
3. **检查配置** → JSON 对不对，Key 对不对
4. **问社区** → 还是不行求助

按照这个顺序来，绝大多数问题你自己就能解决。
