# 出了问题怎么办？排错思路比具体命令重要

用 OpenClaw，就像用任何复杂工具，偶尔肯定会出问题。出了问题不要慌，按照这个思路一步步查，90% 的问题你自己就能解决。

## 排错第一步：看日志

OpenClaw 有完整的日志，绝大多数问题，日志里都写明白了。

**日志在哪里看：**

- 如果你是用 daemon 模式运行，日志在 `~/.openclaw/logs/` 下面
- 如果你是在终端运行，直接看终端输出就行了

**看日志看什么：**
- 找 `error` 或者 `Error` 关键字，看错误说明是什么
- 看在哪一步出错了，是安装错了还是配置错了还是运行错了

**记住：** 日志是最好的老师，它比我教你任何方法都准。

## 排错第二步：搜一搜

拿到错误信息，复制关键字，去 GitHub 搜一搜，去 Google/百度搜一搜，大概率已经有人碰到过同样问题，也有人给出解决方法了。

**搜索技巧：**
- 直接复制错误的关键句子去搜
- 加上 `OpenClaw` 关键字，更准确

90% 的常见问题，搜一下就有答案了。

## 排错第三步：检查配置

很多问题都是配置错了：

### 常见配置错误：
1. **JSON 格式错了**：逗号漏了，括号不配对，引号不配对，JSON 解析错了，整个配置就加载失败
   - 解决方法：把你的配置复制到网上的"JSON校验工具"，一检查就知道哪里错了

2. **API Key 错了**：多复制了空格，少复制了字母，就是不对
   - 解决方法：删掉重新复制，仔细点

3. **模型 ID 错了**：模型 ID 和厂商对不上，找不到模型
   - 解决方法：对照厂商文档看看 ID 对不对

4. **权限错了**：文件权限不对，API Key 权限不对
   - 解决方法：检查 `~/.openclaw` 权限，检查 API Key 在厂商后台有没有开对权限

## 排错第四步：问社区

如果你搜了还是解决不了，去社区问：

- OpenClaw GitHub Issues：https://github.com/openclaw/openclaw/issues
- Discord 社区：官方有 Discord 频道，很多人在里面聊天

问问题记得：
- 贴上你的日志错误
- 说清楚你做了什么，到哪一步出问题了
- 说清楚你的系统是什么（macOS/Linux/Windows）
- 这样别人才能帮你

## 记住：方法比具体命令重要

OpenClaw 更新快，今天的具体命令明天可能就变了，但是排错思路永远不变：

1. **先看日志** → 错误信息是什么
2. **再搜一搜** → 别人碰到过吗
3. **检查配置** → 格式对不对，key对不对
4. **问社区** → 还是解决不了，找人帮忙

按照这个顺序来，绝大多数问题你自己就能解决。
