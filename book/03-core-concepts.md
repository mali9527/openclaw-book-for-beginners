# 五个核心组件，用大白话讲明白

上一章我们说了，AI 智能体就是能自己帮你干活的 AI。它到底是怎么做到的？其实就是五个核心组件拼起来的。

这一章我把每个组件都用生活化类比讲清楚，你看完就明白智能体到底怎么工作了，一点都不深奥。

## 1. 推理服务 = AI 的大脑

**类比：** 就像你我的大脑，负责思考、说话、回答问题。

大模型本身就是一大堆参数文件，存在你的硬盘上，没法直接用。推理服务就是把这个大模型加载到显卡内存里，接收你的问题，一点点生成回答。

简单说：
- 你提问 → 推理服务算一算 → 给你输出回答
- 就这么简单

你不需要自己搭推理服务，现在大部分人都是用 OpenAI、Anthropic 或者国内厂商提供的推理服务，你给点钱就行，人家给你准备好了。

## 2. 记忆机制 = AI 的笔记本

**类比：** 就像你记笔记，重要的东西记下来，以后要用的时候翻一翻。

你可能会问，AI 不是什么都记得吗？其实不是。推理服务每次处理你的问题都是"无状态"的——处理完就忘了，下次问它它还是什么都不知道。

而且大模型一次能处理的文字是有限的，不能把你一年前说过的话都塞进去。所以需要一套"记忆机制"来管理：

- **短期记忆**：最近几轮对话原封不动保留，保证这次对话连贯
- **长期记忆**：时间久了的对话，会用一个小模型提炼成摘要，或者提取出你的特征（比如"用户喜欢简洁回答"、"用户住在上海"），存在文件里，下次要用的时候再找出来

这样 AI 就真的能"记住你"，越用越懂你。

## 3. RAG 检索增强 = AI 的参考书

**类比：** 就像你写文章的时候，遇到不确定的东西，先翻一翻参考书，再下笔写。

大模型有个问题：它的知识来自训练数据，训练完就"冻结"了，不知道之后发生的新闻，也不知道你自己电脑上的私人文件。

RAG 就是解决这个问题的：

> 先搜资料，找到相关内容，再把资料拼给 AI，AI 参考资料回答你。

这样回答就准多了，不会一本正经地胡说八道。

比如你问它"我去年做的那个项目进度怎么样了"，它先在你的记忆里找到去年那个项目的记录，拼给大模型，大模型就能准确回答你了。

## 4. MCP 工具调用 = AI 的手

**类比：** 就像你的手，能拿东西、点鼠标、改文件。

有了大脑（推理）、笔记本（记忆）、参考书（RAG），AI 还是只能待在服务器里说说话，没法真的动手改变什么。

MCP 就是一套标准，告诉 AI 怎么调用外面的工具：

- 要搜索网络 → 调用搜索工具
- 要读文件 → 调用文件读取工具
- 要发邮件 → 调用邮件工具
- 要运行命令 → 调用终端工具

有了手，AI 才能真的动手干活。

MCP 就是一套统一的规矩，不管是什么工具，AI 都能用同样的方式调用它。

## 5. Skills 技能 = AI 的菜谱

**类比：** 就像菜谱，告诉你做一道菜放什么调料，第一步干什么第二步干什么。

MCP 解决了"AI 能不能调用工具"的问题，但是 AI 不知道"什么时候用什么工具，按什么顺序用"。就像你有了所有食材调料，还是需要菜谱告诉你怎么做菜。

Skills 就是干这个的：

它把特定场景下，**工具的使用顺序、执行逻辑、注意事项**都写好了，AI 照着做就行了。

举个例子：
- 客服处理工单：先读用户诉求 → 查用户历史记录 → 给解决方案 → 解决不了转人工 → 关闭工单
- 这个流程就是一个 Skill，AI 跟着走就不会错

Skills 还有一个好处：它是"渐进式加载"的，你用得到哪个技能就加载哪个，不会一下子把几百个技能都塞给 AI，AI 调用起来更准确。

## 什么时候就变成完整的 AI 智能体了？

当一个 AI 同时拥有这五个组件：

| 组件 | 作用 |
|------|------|
| 推理服务 | 思考回答 |
| 记忆 | 记住你的偏好和历史 |
| RAG | 查找外部知识 |
| MCP | 调用工具动手干活 |
| Skills | 按流程完成任务 |

它就变成了一个能**自主行动、完成目标**的 AI 智能体了。
